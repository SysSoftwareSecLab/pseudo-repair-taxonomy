# Pseudo-Repair Taxonomy

**Pseudo-Repair in Human-Driven Iterative LLM Code Refinement**

This repository accompanies the technical report:

Ren, X. (2026). Pseudo-Repair in Human-Driven Iterative LLM Code Refinement: A Taxonomy of Apparent Fixes as an Open Technical Report (1.0). Zenodo. https://doi.org/10.5281/zenodo.19671315

## What is Pseudo-Repair?

Pseudo-Repair describes apparent fixes produced during iterative 
LLM code refinement that create the impression of successful 
remediation without substantively closing the underlying 
security vulnerability.

### Four-class taxonomy:
- **True Repair** — vulnerability substantively closed
- **Pseudo-Repair** — apparent fix, vulnerability remains
- **Vulnerability Mutation** — new vulnerability introduced
- **Regression** — functionality broken during repair

### Pseudo-Repair subtypes:
- PR-SS: Surface Suppression
- PR-SN: Scope Narrowing  
- PR-VM: Vulnerability Migration
- PR-SR: Semantic Regression
